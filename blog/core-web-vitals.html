Core Web Vitals: Guía Definitiva 2024

Google ahora rankea webs basándose en la experiencia del usuario, y los Core Web Vitals son la medida oficial. Si tu web es lenta o inestable visualmente, estás perdiendo posiciones. En esta guía aprenderás a optimizar las 3 métricas clave que Google evalúa.

¿Qué son los Core Web Vitals?

Son 3 métricas que Google usa para medir la experiencia del usuario:

LCP - Largest Contentful Paint

Qué mide: Cuánto tarda en cargar el elemento visual más grande de tu página.

Objetivo: Menos de 2.5 segundos

Por qué importa: Los usuarios abandonan si la página tarda en mostrar contenido.

FID - First Input Delay (ahora INP)

Qué mide: Cuánto tarda tu web en responder cuando el usuario hace click.

Objetivo: Menos de 100 milisegundos

Por qué importa: Una web que no responde frustra al usuario.

CLS - Cumulative Layout Shift

Qué mide: Cuánto se mueven los elementos mientras carga la página.

Objetivo: Menos de 0.1

Por qué importa: ¿Alguna vez intentaste hacer click y la página se movió? Eso es CLS alto.

Cómo Optimizar LCP (Velocidad de Carga)

1. Optimiza Imágenes

  • Usa formatos modernos: WebP o AVIF
  • Comprime sin perder calidad: TinyPNG, ImageOptim
  • Implementa lazy loading para imágenes fuera del viewport
  • Usa srcset para servir imágenes según el dispositivo

2. Mejora el Servidor

  • Usa un CDN (Cloudflare, CloudFront)
  • Actualiza a HTTP/3
  • Optimiza TTFB (Time to First Byte) con caché del servidor

3. Optimiza el CSS y JavaScript

  • Minimiza archivos CSS y JS
  • Elimina JavaScript no usado
  • Carga JS de forma asíncrona o defer
  • Inline critical CSS

Cómo Mejorar FID/INP (Interactividad)

Principales culpables del FID alto:

  • JavaScript pesado: Divide el código en chunks más pequeños
  • Tareas largas: Divide procesos grandes en tareas más pequeñas
  • Third-party scripts: Carga Google Analytics, Facebook Pixel, etc. de forma asíncrona

Técnicas de optimización:

  • Code splitting: carga solo el JS necesario por página
  • Web Workers: mueve operaciones pesadas fuera del thread principal
  • Throttling y debouncing para eventos como scroll

Cómo Reducir CLS (Estabilidad Visual)

Causas comunes de CLS:

  • Imágenes sin dimensiones especificadas
  • Anuncios que se insertan dinámicamente
  • Fuentes web que causan FOIT/FOUT
  • Contenido inyectado por JavaScript

Soluciones:

  • Reserva espacio: Define width y height en todas las imágenes
  • Aspect-ratio CSS: Mantiene el espacio mientras carga
  • Preload fuentes: Evita cambios de tipografía
  • Transform en lugar de top/left: Para animaciones

Herramientas para Medir

1. PageSpeed Insights (Google)

La herramienta oficial. Te da puntuación de 0-100 y sugerencias específicas.

2. Lighthouse (Chrome DevTools)

Análisis detallado en tu navegador. Perfecto para desarrollo.

3. Search Console

Informe de Core Web Vitals con datos reales de tus usuarios.

4. WebPageTest

Análisis profundo desde múltiples ubicaciones.

Optimizaciones Rápidas (Quick Wins)

Si tienes poco tiempo, empieza por aquí:

  1. Activa caché del navegador (5 minutos)
  2. Comprime imágenes (30 minutos)
  3. Usa un CDN gratuito como Cloudflare (15 minutos)
  4. Minimiza CSS y JS (20 minutos)
  5. Define dimensiones de imágenes (10 minutos)

Caso Real: De 23 a 95 en PageSpeed

Cliente: Ecommerce de moda
Problema: Puntuación PageSpeed de 23/100
Acciones:

  • Optimizamos 847 imágenes (reducción del 73%)
  • Implementamos lazy loading
  • Migramos a Cloudflare CDN
  • Eliminamos 14 plugins innecesarios
  • Minimizamos CSS y JS

Resultado: Puntuación 95/100, tiempo de carga de 5.2s a 1.8s
Impacto SEO: Posiciones promedio mejoraron 12 lugares en 2 meses

Errores Comunes

1. Optimizar solo para desktop: Google evalúa principalmente móvil

2. Ignorar el hosting: Un hosting barato puede arruinar todas tus optimizaciones

3. Demasiados plugins: En WordPress, 30+ plugins suelen causar problemas

4. No medir datos reales: PageSpeed da estimaciones, Search Console datos reales

WordPress: Plugins Recomendados

  • WP Rocket: Caché y optimización completa (de pago)
  • Imagify: Compresión automática de imágenes
  • Autoptimize: Minimiza CSS, JS y HTML (gratis)
  • Perfmatters: Desactiva scripts innecesarios por página

Monitoreo Continuo

Los Core Web Vitals no son un "arregla y olvida". Necesitas monitoreo continuo:

  • Semanal: Revisa Search Console
  • Mensual: Análisis detallado con PageSpeed Insights
  • Después de cambios: Siempre testea antes de lanzar

Conclusión

Los Core Web Vitals son fundamentales para el SEO en 2024. Google premia las webs rápidas y penaliza las lentas. La buena noticia es que optimizarlos mejora tanto tu SEO como la experiencia real de tus usuarios.
